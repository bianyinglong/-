这是第一个文件 撤销的几种情况： 1. 用缓存区中的文件覆盖工作目录中的文件 git checkout 文件名 作用：此时工作目录中的响应文件会恢复到最近一次添加到缓存区中的状态 但缓存区中的文件依然不变 2. 从缓存区中删除文件 git rm --cached 文件名 作用：从缓存区中删除文件，表示文件不在被git追踪管理 但工作目录中依然存在该文件 3. 恢复到指定的某个版本 git reset --hard 提交编号 作用：恢复到指定版本，工作区的文件和指定版本的文件一致（工作区的一些文件可能会被删除） 分支： 切换分支之前一定要先提交当前分支的文件，否则会产生混乱 删除钥匙串中的用户名和密码后的提交
